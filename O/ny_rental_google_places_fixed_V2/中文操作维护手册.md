# NY Rental Map 中文操作维护手册

本文档只写日常操作、数据更新、维护规则和注意事项。项目主目录：

```txt
F:\F\ny\O\ny_rental_google_places_fixed_V2
```

## 1. 启动网页

打开 PowerShell，进入项目目录：

```powershell
cd F:\F\ny\O\ny_rental_google_places_fixed_V2
npm run dev
```

本地打开：

```txt
http://localhost:5503
```

后台运营面板：

```txt
http://localhost:5503/admin
```

当前测试密码是 `123456`。正式上线前必须改 `.env.local` 里的 `ADMIN_PASSWORD` 和 `ADMIN_SESSION_SECRET`。

## 2. 数据流怎么理解

这个项目现在有三层数据：

1. 本地 CSV：`data/` 文件夹里的表。
2. Google Sheet：给中介维护、给你审核整理的线上表。
3. 网页缓存：`.data/google-sheets-cache.json`，网页优先读取它。

日常建议以 Google Sheet 为主。本地 CSV 主要作为备份和初始化来源。

## 3. Google Sheet 表结构

Google Sheet 必须有这些 tab，名字要完全一致：

```txt
buildings
units
photos
nearby_pois
contacts
agents
data_sources
change_log
```

核心表说明：

- `buildings`：楼盘信息，一行一个楼。
- `units`：户型和房源信息，一行一个可租户型。
- `photos`：楼图、户型图、室内图。
- `nearby_pois`：附近设施，包括餐厅、超市、咖啡、地铁。
- `contacts`：楼盘联系人或中介联系人。
- `agents`：合作中介人员。
- `data_sources`：数据来源记录。
- `change_log`：重要修改记录。

每个楼和每个户型都应该维护这些可信度字段：

```txt
last_updated_at
source_name
source_url
price_status
fee_status
availability_status
availability_checked_at
contact_id
updated_by
internal_notes
```

推荐状态值：

```txt
verified
provided
needs_confirmation
unknown
```

## 4. 本地覆盖 Google Sheet

用途：第一次初始化，或你确认本地 CSV 是最新版本，要把线上 Sheet 全部重置。

```powershell
cd F:\F\ny\O\ny_rental_google_places_fixed_V2
npm run setup:sheets
```

注意：这个命令会覆盖 Google Sheet 里的 8 张表。中介如果已经在线上改过内容，不要随便运行。

## 5. Google Sheet 覆盖网页

用途：中介或你在 Google Sheet 更新后，让网页显示最新数据。

方式一：打开后台：

```txt
http://localhost:5503/admin
```

登录后点击同步按钮。

方式二：调用接口：

```txt
POST /api/admin/sync
Header: x-admin-sync-token: .env.local 里的 ADMIN_SYNC_TOKEN
```

同步成功后会更新：

```txt
.data/google-sheets-cache.json
```

网页会优先读取这个缓存。

## 6. Google Sheet 覆盖本地 CSV

用途：线上 Sheet 已经是最新，要把本地 `data/` 文件夹也同步回来。

```powershell
cd F:\F\ny\O\ny_rental_google_places_fixed_V2
npm run pull:sheets
```

这个命令会先自动备份旧 CSV 到：

```txt
data/backups/google-sheet-pull-时间戳/
```

然后写入：

```txt
data/buildings.csv
data/units.csv
data/photos.csv
data/nearby_pois.csv
data/contacts.csv
data/agents.csv
data/data_sources.csv
data/change_log.csv
```

注意：`nearby_pois.csv` 是新的合并附近设施表。网页本地模式会优先读它；如果没有它，才会读旧的 `building_google_nearby_pois_500m.csv` 和 `community_pois.csv`。

## 7. 推荐日常更新流程

### 中介给你更新房源

1. 中介在 Google Sheet 里更新 `buildings`、`units`、`photos` 或 `contacts`。
2. 你检查价格、费用、可租状态和来源链接。
3. 填好可信度字段，尤其是 `last_updated_at`、`source_name`、`source_url`、`price_status`、`fee_status`、`availability_checked_at`。
4. 去 `/admin` 点同步。
5. 打开前台检查楼盘详情、户型详情、附近设施、对比和咨询按钮。

### 你要保留一份本地备份

```powershell
npm run pull:sheets
```

### 你要重新初始化一张空的 Google Sheet

先确认本地 CSV 是你要保留的版本，然后运行：

```powershell
npm run setup:sheets
```

## 8. 每 4 小时更新一次

如果网站部署到线上，可以设置定时任务每 4 小时请求：

```txt
POST https://你的域名/api/admin/sync
Header: x-admin-sync-token: ADMIN_SYNC_TOKEN
```

这样 Google Sheet 更新后，网页缓存会自动刷新。

本地电脑不适合长期做定时同步，因为电脑关机、断网、端口被占用都会中断。

## 11. 添加照片和户型图

在 `photos` 添加：

```txt
photo_id
building_id
unit_id
photo_type
photo_url

source_last_checked
```

常用 `photo_type`：

```txt

```

如果是楼盘图，`unit_id` 可以空。  
如果是某个户型图，必须填对应 `unit_id`。

## 12. 附近设施维护

`nearby_pois` 里的 `poi_type` 建议只用：

```txt

grocery
coffee

```

每行至少填写：

```txt

building_id
building_name
poi_type
name
address
distance_meters
lat
lng
source
source_last_checked
```

附近设施建议每个月更新一次，不需要每天更新。

## 13. 运营面板怎么看

进入：

```txt
http://localhost:5503/admin
```

重点看：

- visits：访问量。
- school clicks：学校点击。
- building clicks：楼盘点击。
- unit clicks：户型点击。
- share clicks：分享点击。
- contact clicks：咨询点击。
- lead submissions：留资数量。
- conversion rate：转化率。
- popular schools：热门学校。
- popular buildings：热门楼盘。
- listing confidence gaps：哪些房源可信度字段不完整。

这些数据以后可以作为向中介收费的证据。

## 14. 安全注意事项

- `.env.local` 不能发给别人，不能上传 GitHub。
- Google Sheet 不要公开发布到 web。
- Sheet 只分享给 service account 和你信任的人。
- Google API key 要限制 API 类型、限制域名或服务端环境、设置预算提醒。
- `ADMIN_PASSWORD` 上线前一定要改。
- `ADMIN_SYNC_TOKEN` 不要截图给别人。
- 中介可以编辑 Sheet，但不要让中介碰 `.env.local`。

## 15. 常见问题

### 端口被占用

如果 `npm run dev` 提示 `EADDRINUSE: address already in use :::5503`，说明 5503 已经有一个服务在跑。可以直接打开：

```txt
http://localhost:5503
```

现在 `npm run dev` 会先检测 5503 是否已经有服务。如果已经在运行，它会提示网页地址，不会再重复启动一个服务。

如果页面按钮没反应、控制台出现 `/_next/static/chunks/...page.js 404`，通常是旧的 Next 开发服务没有重启。关掉旧服务后重新运行：

```powershell
npm run dev
```

然后在浏览器里按 `Ctrl + F5` 强制刷新。

### 网页没有显示 Google Sheet 的更新

先去 `/admin` 点同步。  
如果还不显示，确认 `.data/google-sheets-cache.json` 是否更新。

### Google Sheet 同步失败

检查：

- Google Sheets API 是否启用。
- Sheet 是否分享给 `GOOGLE_SERVICE_ACCOUNT_EMAIL`。
- service account 权限是否是 Editor 或 Viewer。
- `.env.local` 里的 `GOOGLE_SHEET_ID` 是否正确。

### 拉回本地后想恢复旧 CSV

去这个文件夹找最近一次备份：

```txt
data/backups/
```

把里面的 CSV 覆盖回 `data/` 即可。

## 16. 最推荐的维护原则

Google Sheet 是运营主数据，本地 CSV 是备份。  
中介只负责提供和更新信息，你负责审核、整理、同步、发布。  
任何价格、费用、可租状态，都要有来源、更新时间和确认状态。
